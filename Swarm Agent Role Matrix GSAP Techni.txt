Swarm Agent Functional Definitions In high-performance motion systems, the swarm must operate as a unified pipeline to guarantee 60fps/120fps fluidic motion and total accessibility. These agents do not function in isolation; they are architected to ingest and emit stateful data within the Windsurf environment. 1. Discovery Agent (The Librarian) : Must enforce environment readiness and dependency integrity. It generates a dependency-graph.json manifest that maps required GSAP core files and plugins (e.g., ScrollTrigger, MorphSVG). It mandates GSAP 3.13+ to leverage the "No Charge" license and modernized SplitText architecture. 2. Deciphering Agent (The Director) : Mandates the Imperative "Director" model to eliminate declarative state-conflict jank. It translates creative intent into granular property commands, prioritizing xPercent, yPercent, and autoAlpha to bypass the browser’s Critical Rendering Path bottlenecks. 3. Classification Agent (The Auditor) : Enforces performance and ethical benchmarking. It audits the animation lifecycle for "Layout Thrashing" and mandates gsap.matchMedia() for vestibular disorder safety. It is specifically tasked with detecting nested interactive elements within text targets to prevent screen-reader fragmentation. 4. Creation Agent (The Orchestrator) : Manages timeline choreography and multi-renderer synchronization. It ingests the dependency-graph.json from the Discovery Agent to initialize complex gsap.timeline() sequences. It governs the master gsap.ticker to sync JavaScript logic with WebGL/Three.js render loops. -------------------------------------------------------------------------------- #### 2. Role-Specific Technical Task Specifications ##### 3.1 Discovery: Installation and Dependency Management * Version Enforcement : Mandate GSAP 3.13+ for all new projects. Older versions must be refactored to comply with the 2025 Standard "No Charge" License. * Plugin Registration : Mandatory gsap.registerPlugin() for all identified dependencies (ScrollTrigger, SplitText, etc.) to prevent tree-shaking from purging critical motion logic. * Module Strategy : Detect environment type (ESM vs. UMD) and serve optimized files from the /minified/ or /esm/ directories to minimize the Critical Rendering Path footprint. ##### 3.2 Deciphering: Architectural Logic and Property Shorthands * Layer Promotion Shorthands : Prohibit use of top, left, margin, or width for motion. Mandate x, y, scale, and rotation. * Percentage Optimization : Always prefer xPercent and yPercent for responsive movement to ensure the Compositor Thread handles the workload without reflows. * Visibility Management : Enforce autoAlpha over opacity to automate visibility: hidden toggles, preventing "ghost" pointer-events and optimizing GPU layer memory. ##### 3.3 Classification: Performance & Accessibility Benchmarking * Vestibular Disorder Protocols : Must implement gsap.matchMedia() to monitor prefers-reduced-motion. If detected, agents must simplify motion to opacity fades and eliminate large swiping/zoom transforms. * SplitText A11y Nuance : Default to aria: "auto". However, if nested interactive elements (e.g., , , ) are detected, the agent must override to aria: false and implement a screenreader-only (sr-only) duplicate pattern to preserve semantic functionality. * Stability Management : Activate lagSmoothing(500, 33) to maintain synchronization during CPU computational spikes, preventing clumping of staggered animations. ##### 3.4 Creation: Timeline Choreography and Shader Integration * Master Timeline Pattern : Mandate the modular return of timelines into a single master instance using position parameters (<, +=, -=) for surgical timing. * WebGL Rendering Loop : For Three.js or custom shaders, agents must sync the renderer by binding to gsap.ticker.add(stage.render.bind(stage)). * Uniform Animation : Animate shader uniforms (e.g., uTime, uMixFactor, uRippleProgress) directly to offload complex visual transformations to GPU parallel processing cores. -------------------------------------------------------------------------------- #### 4. The convergence of WebGL (Three.js) and GSAP-powered Webflow Interactions defines the new standard for visual engineering. Swarm agents must handle the integration of procedural GPU effects (noise, liquid reveals, liquid ripples) that remain modular. This ensures that the generated motion systems are robust, performant, and fully accessible to users across the technical spectrum.
